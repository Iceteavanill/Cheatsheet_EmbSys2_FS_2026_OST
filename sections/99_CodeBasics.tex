\subsection{Calling C library code from C++ applications}

To make a C library callable from C++ code the following template must be used:

\begin{minipage}[t]{0.5\columnwidth}
\begin{lstlisting}
// LIB.h
#ifndef INCLUDE_LIB_H_ // includeguards
#define INCLUDE_LIB_H_
#ifdef __cplusplus // check if c++ compiler
extern "C"{ // Wrap all code with "extern"
#endif

int libfunc(int param);
// ...
#ifdef __cplusplus
};
#endif // (#ifdef __cplusplus)
#endif // (#ifndef INCLUDE_LIB_H_)
\end{lstlisting}
\end{minipage}
\begin{minipage}[t]{0.5\columnwidth}
\begin{lstlisting}
// LIB.c
# include "LIB.h"
int libfunc(int param){
    return param & 1;
}
\end{lstlisting}
Because the function is wrapped with \verb|extern|, nothing has to be done in the .c file.
The compiler was already told that the function has the C calling conventions.
\end{minipage}

\subsubsection{Name mangling}

Name mangling roughly means that the function name is altered in the compiling step.
That is done to resolve namespace issues if 2 functions have the same name but different namespaces.
Because this is only applied in C++, the compiler must adjust the naming scheme if a C library should be called.

\subsection{memcpy}
memcpy copies a fixed number of bytes from one memory location to another.
\begin{lstlisting}
memcpy(destination, source, number_of_bytes);
int a = 42;
int b;
memcpy(&b, &a, sizeof(int)); // b now holds 42
\end{lstlisting}

\subsection{Unit Tests}
To test your implementation, you can use the \texttt{assert.h} library in C. This allows you to write simple unit tests to verify that your functions behave as expected.
The tests should be written to check for both typical and edge cases.
\begin{lstlisting}
  float a = 1.0f;        // test simple add function
  float b = 2.0f;
  float expected = 3.0f;
  float result = Add(a, b);
  assert(fabsf(result - expected) < 0.001f && "Add normal case failed");
  printf("Test 1 passed\n");
\end{lstlisting}

\subsection{Difference CRC and checksum}
\begin{minipage}[t]{0.69\linewidth}
    \textbf{checksum}\\
    Typically done via 2's complement.
    The address and data is added and sent as the inverted 2's complement as well.
    On the receiver, the same calculation is made and the 2 values added.
    If zero results, the transmission was successful.
    Easy way to detect errors in data transmission.
\end{minipage}
